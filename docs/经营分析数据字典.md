# 经营分析数据字典

本文以当前扩展实现为准，定义经营分析的报告身份、数据源、规范化字段、35 项核心值、风险和完整性语义。金额原值单位为元，页面和离线 HTML 默认显示为万元。

## 1. 报告身份与范围

正式报告由以下字段唯一确定：

| 字段 | 含义 |
|---|---|
| `departmentId` | 组织树部门 ID；`all` 表示全部有效部门 |
| `departmentName` | 展示名称，不参与项目归属匹配 |
| `startDate/endDate` | 闭区间，格式 `YYYY-MM-DD` |
| `configVersion` | 项目筛选配置的稳定 hash |
| `policyVersion` | 风险阈值配置的稳定 hash |
| `capturedAt` | 数据捕获 ISO 时间 |

这些字段只标识当前页面中的正式报告，不作为持久化 key。每次查询和刷新都会重新请求实时数据。同一筛选维度内为 OR，不同维度间为 AND；最后按 `project.projectDept === department.id` 精确分组。

默认筛选：属性不限，分类 `J/Z`，一级状态 `10/20/50`，执行类型不限，并且正式范围仅保留本期 `sum(realHour) > 0` 的项目。默认日期为按中国时区计算的最近一个完整周一至周日。

## 2. 数据源

| 来源 | Endpoint / queryName | 关键字段 | 用途 |
|---|---|---|---|
| 组织树 | `/rest/org/tree/unitTree` | `id`, `text`, `attributes.privLevel` | 保留 `privLevel !== "1"` 的部门；定义可选范围 |
| 项目主数据 | `/rest/project/ProjectInfoService/query`, `queryList` | 项目、部门、PM、合同、预算、成本、进度 | 全分页后本地筛选；累计经营指标 |
| 日报 | `/rest/project/taskDetailService/query`, `queryTaskDetail` | `projectId`, `taskDate`, `realHour`, `cost` | 本期/上期投入和成本 |
| WBS | `/rest/project/ProjectPlanDetailService/query`, `queryVer` | `costLevel`, `planEndTime`, `actualEndTime` | PV、EV、SPI 和区间产服 EV |
| 里程碑 | 同上，`queryLandmark` | 节点、计划/实际日、`confirmStatus` | 月完成、逾期和未来 7 天 |
| 周报列表 | `/rest/project/WkReportService/query`, `queryByProjectId` | 周报 ID、周期、更新时间 | 选择与报告区间相交的最新有效周报 |
| 周报详情 | `/rest/project/queryByProjectInfosService/query`, `queryByProjectInfo` | 本期总结、下期计划、执行行 | 周期执行表和下期计划人天 |
| 项目回款 | `/rest/project/queryProjectPlanService/query`, `invoicePlanDetailList` | 项目 ID、合同、计划日/金额、实收 | 当月回款和逾期未回 |
| 当月回款补充 | `/rest/contract/queryReceivedPlanService/query`, `queryMyList` | 合同、预计日、计划/实收金额 | 仅唯一合同匹配时补充当前正式项目 |

所有分页使用响应 `recordsTotal/total` 或短页结束条件，不设 2000 条或固定页数上限。接口复用当前 JXPMO 登录会话，不保存 Cookie。

| `projectNo` | `projectNo` | 展示编码 |
| `projectName` | `projectName` | 必填 |
| `contractNo` | `contractNo` 或 `contractCode` | 回款补充的唯一合同关联 |
| `attribute` | `attribute` | `C/P/A/S` |
| `classification` | `classification` | `J/R/X/S/Z/W/Y/Q` |
| `currStatus` | `currStatus` | `10/20/30/40/50/0/60` |
| `outsourcing` | `outsourcing` | `01` 执行，`02` 外包执行 |
| `projectDept` | `projectDept` | 必填，部门 ID 精确匹配 |
| `projectDeptName` | `projectDeptName` | 仅展示/非当前组织部门诊断 |
| `projectManager` | `projectManager` | PM 标识 |
| `projectManagerName` | `projectManagerName` | PM 展示名 |
| `isCreateWkReport` | `isCreateWkReport` | 判断周报是否适用 |
| `contractAmount` | `contractAmount` | 合同总额，PM 明细展示 |
| `subcontractAmount` | `subcontractAmount` | 软件与服务合同金额 `revenue` |
| `estiExeuCost` | `estiExeuCost` | 执行预算 BAC |
| `realExeuCost` | `realExeuCost` | 累计实际成本 AC |
| `realWorkload` | `realWorkload` | 累计人天；人年=`realWorkload/261` |
| `planCompleteSchedule` | `planCompleteSchedule` | 进度百分数，计算前除以 100 |

接口成功时，`null`、空字符串或缺失的可选业务数值字段规范化为 0。无法解析的非空金额、日期或必填标识会产生 schema error；技术失败不会静默转为 0。

## 4. 基础公式

| 指标 | 公式 |
|---|---|
| `progress` | `planCompleteSchedule / 100` |
| `revenue` | `subcontractAmount` |
| `BAC` | `estiExeuCost` |
| `AC` | `realExeuCost` |
| `CR` | `revenue * progress` |
| 累计 `EV` | `BAC * progress` |
| `CPI` | `EV / AC` |
| `CCPI` | `CR / AC` |
| `EAC` | `BAC / CPI` |
| 人年 | `realWorkload / 261` |
| 整体人均产值 | `CR / 人年` |
| 剩余预算 | `max(BAC - AC, 0)` |
| 预算执行率 | `AC / BAC` |
| 本期投入人天 | `sum(daily.realHour) / 8` |
| 本期投入成本 | `sum(daily.cost)` |
| 投入环比 | `(本期人天 - 上期人天) / 上期人天` |
| 成本环比 | `(本期成本 - 上期成本) / 上期成本` |

所有 ratio 先聚合原值再计算。分子、分母已知且分母大于 0 时正常计算；已知分母为 0 时强制为 0；依赖来源技术失败而未知时为 `null/未获取`。

### 4.1 WBS 截止规则

`costLevel` 是 WBS 权重/产值原值：

- 月 PV：计划完成日在结束日期所在自然月内的 `sum(costLevel)`。
- 月 EV：实际完成日在该月内且不晚于 `endDate` 的 `sum(costLevel)`。
- 区间 PV/EV：计划/实际完成日在 `[startDate, endDate]` 内。
- 累计 PV/EV：计划/实际完成日不晚于 `endDate`。
- 月/区间/总 SPI：对应 `EV / PV`。
- 区间产服 EV：`区间EV / BAC * revenue`。
- 区间 CPI：`区间EV / 本期日报成本`。
- 区间 CCPI：`区间产服EV / 本期日报成本`。
- 区间人均产值：`区间产服EV / (本期投入人天 / 261)`。

未来实际完成记录不能穿透历史截止日。

### 4.2 有投入项目

项目只有在本期投入人天大于 0 时才进入正式范围和“有投入项目经营”。上期有投入但本期无投入的项目只进入“投入范围变化”诊断，不参与本期或上期正式指标。

下期计划人天来自相交周报执行行聚合小时除以 8。预算日消耗=`本期日报成本/区间天数`，预计耗尽天数=`剩余预算/预算日消耗`。

## 5. 35 项核心值

报告渲染 34 张卡片、35 个数值；“成本效率”卡同时包含 CPI 和 CCPI。

### 5.1 部门经营概览（13 个值）

| ID | 展示名 | 来源/公式 |
|---|---|---|
| `projectCount` | 项目数 | 正式项目集合去重计数 |
| `revenue` | 软件与服务合同 | `sum(subcontractAmount)` |
| `bac` | BAC | `sum(estiExeuCost)` |
| `ac` | AC | `sum(realExeuCost)` |
| `cr` | CR | 项目 CR 汇总 |
| `ev` | EV | 项目累计 EV 汇总 |
| `perCapita` | 整体人均产值 | 汇总 CR / 汇总人年 |
| `cpi` | CPI | 汇总 EV / 汇总 AC |
| `ccpi` | CCPI | 汇总 CR / 汇总 AC |
| `eac` | EAC | 汇总 BAC / 汇总 CPI |
| `attentionProjectCount` | 需关注项目数 | 至少一个风险的去重项目数 |
| `monthSPI` | 当月 SPI | 有投入项目月 EV / 月 PV |
| `totalSPI` | 总 SPI | 有投入项目累计 EV / 累计 PV |

### 5.2 有投入项目经营（15 个值）

| ID | 展示名 | 来源/公式 |
|---|---|---|
| `activeProjectCount` | 有投入项目数 | 上述判定后的项目数 |
| `inputMd` | 本周/本期投入人天 | 日报小时 / 8 |
| `inputCost` | 本周/本期投入成本 | 日报 `cost` 汇总 |
| `inputDelta` | 投入环比 | 与相邻上期比较 |
| `costDelta` | 成本环比 | 与相邻上期比较 |
| `activeMonthSPI` | 月 SPI | WBS 月 EV / 月 PV |
| `periodSPI` | 区间 SPI | WBS 区间 EV / PV |
| `periodPV` | 区间 PV | 区间计划 WBS 权重 |
| `periodEV` | 区间 EV | 区间实际完成 WBS 权重 |
| `serviceEV` | 区间产服 EV | 区间 EV / BAC * 软件服务合同 |
| `periodCPI` | 区间 CPI | 区间 EV / 日报成本 |
| `periodCCPI` | 区间 CCPI | 区间产服 EV / 日报成本 |
| `periodPerCapita` | 区间人均产值 | 区间产服 EV / 期间人年 |
| `activeRiskItems` | 需关注项 | 风险条目数，不按项目去重 |
| `nextPeriodPlannedMd` | 下周/下期计划人天 | 周报计划小时 / 8 |

### 5.3 里程碑（3 个值）

| ID | 展示名 | 规则 |
|---|---|---|
| `milestonePlanned` | 本月应完成 | 计划日位于结束月的节点数 |
| `milestoneCompletion` | 本月完成率 | `confirmStatus === "2"` 数 / 应完成数 |
| `milestoneOverdue` | 已逾期 | 未完成且计划日早于 `endDate` |

未来 7 天列表为未完成、计划日晚于 `endDate` 且不晚于 `endDate + 7`。

### 5.4 回款（4 个值）

| ID | 展示名 | 规则 |
|---|---|---|
| `invoiceMonthPlan` | 当月计划 | 结束月计划金额汇总 |
| `invoiceReceived` | 已回款 | 结束月实收金额汇总 |
| `invoicePending` | 待回款 | `max(月计划 - 月实收, 0)` |
| `invoiceOverdue` | 逾期未回笔数 | 待回大于 0 且计划日早于 `endDate` |

当月补充记录只有合同号唯一命中正式项目时才纳入；未命中和多重匹配只进入诊断。

## 6. 风险阈值

| 类型 | 默认值 | 触发条件 |
|---|---:|---|
| 低人均产值 | 300000 元/人年 | `0 < perCapita < lowPerCapita` |
| 成本效率低 | CPI 0.85 | `0 < CPI < cpiWarn` |
| 预算超支 | - | `AC > BAC` |
| 严重超支 | 1.15 | `AC/BAC > severeOverrunRatio` |
| SPI 预警 | 0.8 | SPI 低于预警但不低于严重值 |
| SPI 严重 | 0.5 | SPI 低于严重值 |
| 高投入零产出 | 0.5 人天 | 投入大于阈值且区间 EV 为 0 |
| 预算即将耗尽 | 90 天 | 预计耗尽天数不大于阈值 |
| 里程碑逾期 | - | 项目存在逾期里程碑 |
| 回款逾期 | - | 项目存在逾期未回记录 |

阈值变化产生新的 `policyVersion`，用于标识当前报告使用的风险策略。

## 7. 完整性与空值

来源状态包括 `success`、`empty`、`failed`、`notApplicable`：

- `success`：请求和 schema 校验成功。
- `empty`：系统确认合法空结果，相关已知计数/金额可为 0。
- `failed`：网络、会话、HTTP、分页或 schema 失败；依赖指标为 `null/未获取`。
- `notApplicable`：例如项目不要求周报，不计作技术失败。

完整正式报告要求部门/项目范围、当前和上期日报、所有适用项目的 WBS/里程碑/周报/项目回款，以及当月回款补充全部成功或合法空。无论完整或部分，报告都不写入 IndexedDB、查询缓存、快照或指标历史。

部分报告可以查看和导出；`failedRequests` 只在当前页面内存中保留安全的重试 descriptor，禁止 Cookie、JSESSIONID、API Key 和 Authorization 字段。关闭或刷新页面后失败上下文自然丢弃。

期间比较按筛选周期实时推导紧邻上期，不读取历史报告。全部部门由同一次实时结果按部门分组、项目 ID 去重并重新计算比率。

## 8. 导出字段边界

离线 HTML 包含正式报告的卡片、项目聚合、PM、预算、周期执行、里程碑、回款和来源状态；不包含 Cookie、API Key、请求头、接口 URL、完整原始响应、人员级日报、人员岗级成本、周执行人员明细和失败重试描述。临时项目选择不会进入导出。
